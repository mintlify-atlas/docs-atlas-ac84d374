## Init Class

**Location:** main.py:23

**Purpose:** Initializes the application environment and loads board metadata.

### Initialization Steps

<Steps>
  <Step title="Test Dependencies">
    If `make_thumbnails` is enabled, verifies FFmpeg and ImageMagick are installed.
  </Step>
  
  <Step title="Load boards.json">
    Loads board metadata from cache or fetches from 4chan API (`https://a.4cdn.org/boards.json`).
  </Step>
  
  <Step title="Filter Boards">
    Identifies which boards have archive support and stores in `configs.boards_with_archive`.
  </Step>
</Steps>

## Filter Class

**Location:** filter.py:23

**Purpose:** Applies content filtering rules and manages media downloads.

### Properties

| Property | Type | Description |
|----------|------|-------------|
| `board` | str | Board name |
| `tid_2_thread` | dict[int, dict] | Filtered threads to archive |
| `tid_2_posts` | dict[int, list[dict]] | Thread posts for media filtering |
| `full_pids` | set[int] | Post IDs to download full media |
| `thumb_pids` | set[int] | Post IDs to download thumbnails |
| `banned_hashes` | set[str] | Media hashes marked as banned |

### Methods

<AccordionGroup>
  <Accordion title="filter_catalog()" icon="filter">
    **Location:** filter.py:46
    
    Filters catalog threads based on configuration rules.
    
    ```python
    filter = Filter(fetcher, db, 'g', state)
    filter.filter_catalog(catalog)
    # filter.tid_2_thread now contains only threads that pass filters
    ```
    
    **Filters Applied:**
    1. Minimum character requirements (`op_comment_min_chars`, `op_comment_min_chars_unique`)
    2. Blacklist patterns (skip if matched)
    3. Whitelist patterns (require if specified)
    4. Last modified timestamp (skip unmodified threads)
    
    **Side Effects:**
    - Updates `state.thread_cache` with thread timestamps
    - Prunes old threads from state
  </Accordion>
  
  <Accordion title="should_archive()" icon="question">
    **Location:** filter.py:88
    
    Determines if a thread should be archived based on rules.
    
    ```python
    if filter.should_archive(subject="Linux thread", comment="How do I install Arch?"):
        # archive this thread
    ```
    
    **Filter Priority:**
    1. Character minimums (hard requirements)
    2. Blacklist (excludes even if whitelisted)
    3. Whitelist (includes only if matched)
    4. Default (archive everything if no filters)
  </Accordion>
  
  <Accordion title="download_media()" icon="download">
    **Location:** filter.py:196
    
    Downloads media files based on configuration.
    
    ```python
    filter.download_media(tid_2_posts, pid_2_post)
    ```
    
    **Process:**
    1. Calls `get_pids_for_download()` to determine what to download
    2. Downloads full media for PIDs in `full_pids`
    3. Downloads thumbnails for PIDs in `thumb_pids`
    4. Generates thumbnails from full media if `make_thumbnails` enabled
  </Accordion>
  
  <Accordion title="get_pids_for_download()" icon="list-check">
    **Location:** filter.py:154
    
    Determines which posts need media downloads.
    
    **Checks:**
    - Config patterns (`dl_fm_op`, `dl_fm_post`, `dl_fm_thread`, etc.)
    - Banned media hashes
    - File already exists locally
    
    **Populates:** `self.full_pids` and `self.thumb_pids`
  </Accordion>
  
  <Accordion title="is_media_needed()" icon="image">
    **Location:** filter.py:132
    
    Checks if media should be downloaded for a post.
    
    ```python
    if filter.is_media_needed(post, dl_fm_op, MediaType.full_media):
        # download the file
    ```
    
    **Returns False if:**
    - Config pattern doesn't match
    - Media hash is banned
    - File already exists
  </Accordion>
  
  <Accordion title="download_post_file()" icon="file-arrow-down">
    **Location:** filter.py:213
    
    Downloads a single media file.
    
    ```python
    success = filter.download_post_file(post, MediaType.full_media)
    ```
    
    **Features:**
    - Validates file size and MD5 hash
    - Applies image/video cooldowns
    - Updates `{board}_images` table
    - Generates thumbnail if `make_thumbnails` enabled
  </Accordion>
</AccordionGroup>

<Tip>
  The Filter class handles both content filtering (which threads to archive) and media filtering (which images to download). These are separate concerns but managed by the same class for convenience.
</Tip>

## Posts Class

**Location:** posts.py:16

**Purpose:** Fetches thread JSON, detects deletions, and manages post data.

### Properties

| Property | Type | Description |
|----------|------|-------------|
| `board` | str | Board name |
| `tid_2_thread` | dict[int, dict] | Filtered threads from catalog |
| `tid_2_posts` | dict[int, list[dict]] | Maps thread_id → list of posts |
| `pid_2_post` | dict[int, dict] | Maps post_id → post data |

### Methods

<AccordionGroup>
  <Accordion title="fetch_posts()" icon="download">
    **Location:** posts.py:33
    
    Fetches posts for all filtered threads.
    
    ```python
    posts = Posts(db, fetcher, 'g', tid_2_thread, state, catalog)
    posts.fetch_posts(archive)
    ```
    
    **Process:**
    1. Detects threads missing from catalog (deleted/archived/pruned)
    2. Uses catalog incremental updates when possible
    3. Fetches full thread JSON when needed
    4. Detects deleted posts within threads
    5. Marks threads/posts as deleted/archived in database
    
    **Optimizations:**
    - Batch fetches existing PIDs (one query for all threads)
    - Skips full fetches using `can_use_catalog_update()`
    - Lazy-loads archive.json only when needed
  </Accordion>
  
  <Accordion title="can_use_catalog_update()" icon="gauge-high">
    **Location:** posts.py:212
    
    Checks if incremental update is possible.
    
    ```python
    if posts.can_use_catalog_update(thread_data, thread_stats, last_replies):
        # use catalog data instead of fetching thread JSON
    ```
    
    **Requirements:**
    - `last_replies` exists in catalog
    - Thread stats cached from previous loop
    - Reply count increased since last check
    - New replies exactly match catalog's `last_replies`
    
    **Returns:** `True` if safe to use catalog data, `False` otherwise.
  </Accordion>
  
  <Accordion title="process_catalog_update()" icon="arrows-rotate">
    **Location:** posts.py:244
    
    Extracts new posts from catalog's `last_replies`.
    
    ```python
    new_posts = posts.process_catalog_update(tid, last_replies, thread_stats)
    ```
    
    - Filters replies newer than `most_recent_reply_no`
    - Validates each reply against schema
    - Logs number of new posts
  </Accordion>
  
  <Accordion title="classify_missing_thread()" icon="magnifying-glass">
    **Location:** posts.py:165
    
    Determines why a thread is missing from catalog.
    
    ```python
    deletion_type = posts.classify_missing_thread(tid, archive)
    # Returns: DeletionType.archived / .deleted / .pruned / .inconclusive
    ```
    
    **Logic:**
    - If in archive.json → **archived**
    - If recent bump + early page + few replies → **deleted** (by moderator)
    - Otherwise → **pruned** (naturally fell off board)
  </Accordion>
  
  <Accordion title="save_posts()" icon="floppy-disk">
    **Location:** posts.py:304
    
    Saves all fetched posts to database.
    
    ```python

    ```

## Fetcher Class

**Location:** fetcher.py:8

**Purpose:** Handles all HTTP requests with caching and cooldowns.

### Properties

| Property | Type | Description |
|----------|------|-------------|
| `session` | requests.Session | Persistent HTTP session |
| `state` | State | State manager for HTTP caching |

### Methods

<AccordionGroup>
  <Accordion title="fetch_json()" icon="download">
    **Location:** fetcher.py:13
    
    Fetches JSON from a URL with HTTP caching support.
    
    ```python
    data = fetcher.fetch_json(
        url='https://a.4cdn.org/g/catalog.json',
        headers={'User-Agent': '...'},
        request_cooldown_sec=1.0,
        add_random=True
    )
    ```
    
    **Features:**
    - HTTP conditional requests (`If-Modified-Since` header)
    - Returns empty dict on 304 Not Modified
    - Caches `Last-Modified` header in state
    - Applies cooldown after request
    - Handles JSON decode errors gracefully
    
    **Returns:** Dict on success, empty dict on error/304.
  </Accordion>
  
  <Accordion title="sleep()" icon="moon">
    **Location:** fetcher.py:48
    
    Refreshes session between loops.
    
    ```python
    fetcher.sleep()
    ```
    
    Closes and recreates session if `loop_cooldown_sec >= 15` to prevent stale connections.
  </Accordion>
</AccordionGroup>

<Note>
  The Fetcher class implements HTTP caching to reduce bandwidth and respect 4chan's API. If the server returns 304 Not Modified, no data is re-downloaded.
</Note>
